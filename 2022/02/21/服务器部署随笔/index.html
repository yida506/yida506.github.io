Docker

介绍: 本质上是通过打包的方式，将环境啥的都集中到一个地方，避免配置的复杂度，做到随用随取，同时里面相当于包含独立的linux主机类似bin啥的目录

docker pull：

下载网上的镜像，一般选用公用镜像，私有的建立仓库啥的

docker run:
参数 解释
-d 表示启动在后台
-p 8888:8888表示端口
-e 环境(对于jupyter镜像来说，加入JUPYTER_ENABLE_LAB=yes代表使用lab而不是notebook)
-v 给容器挂载存储卷，挂载到容器的某个目录(/主机目录 : /容器目录)
