Framework

Posted on 2022-02-02 Edited on 2022-03-04 In 框架

源码解读

BUFFER组件:

REQUEST_BUFFER:

定义了requests队列和删除队列，用于添加任务URL和删除URL.

ITEM_BUFFER:

将数据添加到数据库中，内置去重等方法。

Core组件

BaseParser：

作为父类，后续继承其中的方法。

Collector:

功能介绍:主要是获取任务，以及分配任务，将request和request对象存入一个队列中

init 部分：

传入：redis_key

继承：多线程threading.Thread类

定义：

  • _thread_stop： boolean

  • _todo_request：定义一个空队列

  • _tab_requests：将redis_key传入requests表
  • _tab_spider_status：将redis_key传入爬取状态表

  • _spider_mark：获取本地ip+时间，应该是导出日志使用
  • _interval ：从任务队列中获取任务到内存队列的间隔
  • _request_count: 每次获取的任务数
  • _is_collector_task:boolean 判断任务是否被收集
  • _first_get_task：boolean 第一次获取任务
run:

使用while循环(使用thread_stop判断)，保证爬虫持续运行。

运行report_node_heartbeat()—>input_data()，出现异常就打印日志.

stop:

用于将run中的while循环终止，结束线程

__input_data:
  1. 获取当前时间
  2. 如果任务队列中的任务数量大于任务队列容量，则不向其中添加任务，并返回。
  3. 获取当前任务队列中的任务数量
  4. 获取一定优先级的任务数量 redis中的方法(不懂)
  5. 首次任去任务重置(不懂)
  6. 获取任务
  7. put_requests 下发任务

不懂redis的原理和心跳等

__report_node_heartbead：

redis相关操作，不懂

__delete_dead_node:

redis相关操作，不懂

__put_requests:

将任务下发到request_dict汇总以获取request和request对象。

将获取的字典传入队列。

HANDLE_FAILED_REQUESTS:

主要功能:

redis相关

PARSER_CONTROL:

主要功能：任务数量统计

init:

传入：collector,redis_key,request_buffer,item_buffer

继承：多线程threading.Thread类

定义：

  • _parses：空列表
  • _collector: 传入
  • _redis_key:传入
  • _requeset_buffer: 传入
  • _thread_stop:boolean
  • _wait_task_time: 0
run:

从collector中获取任务列表，如果不存在，就等待1s，同时将is_show_tip赋值为true，用于后续判断。

将任务列表传入deal_requests中。

deal_requests：

获取request对象和request_redis

解析_parsers列表，解析中间件，获取response结果，对结果进行判断，通过item_buffer入口，失败任务检测

SCHEDULER：

参数:

  • redis_key: 爬虫request及item存放redis中的文件夹
  • thread_count: 线程数，默认为配置文件中的线程数
  • begin_callback: 爬虫开始回调函数
  • end_callback: 爬虫结束回调函数
  • delete_keys: 爬虫启动时删除的key，类型: 元组/bool/string。 支持正则
  • keep_alive: 爬虫是否常驻，默认否
  • auto_start_requests: 爬虫是否自动添加任务
  • batch_interval: 抓取时间间隔 默认为0 天为单位 多次启动时，只有当前时间与第一次抓取结束的时间间隔大于指定的时间间隔时，爬虫才启动
  • wait_lock: 下发任务时否等待锁，若不等待锁，可能会存在多进程同时在下发一样的任务，因此分布式环境下请将该值设置True
  • task_table: 任务表， 批次爬虫传递
init:
  • request_buffer: 初始化RequestBuffer(redis_key)类
  • item_buffer:初始化itembuffer(redis_key,task_table)
  • _collector:初始化Coollector(redis)
  • _parsers:空列表
  • _parser_controls:空列表
  • _parser_control_obj:PaserControl
